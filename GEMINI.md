# PROJECT GUIDES & STANDARDS (GEMINI.md)

## 1. 運用・開発ルール (Operational Integrity)

### 1.1 ディレクトリ構成 (Root Integrity Policy)
ルートディレクトリの乱雑化を防ぐため、配置可能なファイルを以下の種別に限定します。
- **ドキュメント**: `*.md`
- **フロントエンド・エントリポイント**: `*.html`
- **環境設定・保守スクリプト**: `setup.*`, `backup.*`
- **テスト関連**: `test*`, `debug*` (HTML/JS)
- **設定ファイル**: `.gitignore` 等

これら以外のファイル（ユーティリティ、画像、生データ、中間ファイル等）は、必ず `scripts/`, `assets/`, `raw_data/` などの適切なサブフォルダに配置してください。

### 1.2 更新記録 (Update Logging Policy)
プロジェクトの透明性を維持するため、以下のルールを適用します。
- **記録タイミング**: ファイルの新規作成、修正、削除、または構成の変更を行った際。
- **記録場所**: `update_history.md`
- **記載内容**: 日付、変更の概要、および変更の理由や影響範囲。

### 1.3 開発スタックと文字コード
- **文字コード**: 全てのソースコード、データ、スクリプト出力は **UTF-8** で統一します。
- **PowerShell**: 実行時は `[Console]::OutputEncoding = [System.Text.Encoding]::UTF8;` を冒頭に付与し、書き込み時は `-Encoding UTF8` を明示してください。
- **Python**: 日本語を扱う際は UTF-8 エンコーディングを明示してください。
- **強制書き込み**: インデントが自動調整されるツール（write_to_file等）を避ける必要がある場合は、PowerShell の `Set-Content` 等で生のテキストとして書き込んでください。

## 2. システム仕様 (System Specifications)

### 2.1 スキルの並び順 (Display Sorting)
シミュレーターの検索結果やデータベースでのスキル表示は、以下の優先順位でソートされます。
1. **メインカテゴリ優先度**: 武器スキル(`weapon`) > 防具スキル(`armor`) > シリーズスキル(`series`) > グループスキル(`group`)
2. **スキルレベル優先度**: 同じカテゴリ内では、発動レベルが高いものを優先。
3. **内部定義順**: カテゴリ・レベルが同じ場合は、`js/data/skills.js` 内の定義順に従う。
4. **サブカテゴリ優先度**: 攻撃(`attack`) > 会心(`affinity`) > 属性(`element`) > 弾・矢(`ammo`) > その他(`utility`)

### 2.2 シリーズ・グループスキル
- **シリーズスキル**: 同名防具を **2点** または **4点** 装備で発動。名称と発動スキル名が異なる場合がある。
- **グループスキル**: 同名グループスキルを持つ装備を **3点** 装備で発動。システム上は **Lv1固定** として扱う。

### 2.3 装飾品の装着制限
- **武器スロット**: 武器用装飾品のみ装着可能。
- **防具スロット**: 防具用装飾品のみ装着可能。
- **護石スロット**:
  - **RARE 8 (RERA8)**: 第1スロットが `[1]` の場合のみ武器用装飾品を装着可能。それ以外は防具用。
  - **RARE 8 未満**: 全て防具用装飾品のみ。

## 3. UI/UX デザイン規約 (Design System)

### 3.1 カラーパレット
- **背景系**: 基本背景 `#0c111d` / パネル背景 `#141a26`
- **アクセント**: `#ffcc00` (ゴールド/イエロー、ロゴ・重要見出し用)
- **テキスト**: 標準 `#eee` / 補足 `#aaa` / ステータス値 `#ccc`
- **カテゴリ強調**:
  - 武器・テーブル見出し: `#66ccff` (ライトブルー)
  - シリーズスキル [S]: `#ff9900` (オレンジ)
  - グループスキル [G]: `#00cc88` (エメラルドグリーン)

### 3.2 タイポグラフィ
- **フォント**: 原則として **明朝体** (`serif`) を使用。
  - `font-family: 'Times New Roman', "MS Mincho", serif;`
- **数値表示**: 桁を揃えるため必ず **等幅（tabular-nums）** を適用。
  - `font-variant-numeric: tabular-nums;`

### 3.3 レイアウト
- **構成**: 検索結果は 2カラム構成（装備リスト 460px / スキル合計 1fr）。
- **スロット表現**: 背景黒(`#000`)、ボーダーありの四角いボックス形式 `[3]`。
- **装飾品表示**: スロットボックス横に控えめに配置（色: `#b0c4de`）。
